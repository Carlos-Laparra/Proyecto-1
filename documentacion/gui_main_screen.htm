Manual de Usuario

Guía de como utilizar el programa.

1. Elegir si desea ver las instrucciones o ir a crear el tablero directamente.

2. En caso de haber elegido la opción de "Instrucciones" se desplegará un menú con las indicaciones generales del juego, de como llenar el tablero principalmente. Luego de eso hay una única opción que lleva a la creación del tablero.

3.Una vez leídas las instrucciones el usuario deberá ingresar su propio txt. Para ello se toman en cuenta varias consideraciones, como que la cantidad de colores que ingresa no sobrepase la cantidad de listas que puede ingresar, o que los colores no sobrepases la capacidad de las listas, etc. De igual manera, cualquier error cometido se indicará con una ventana emergente. Finalmente cuando el usuario ingrese todo de manera correcta aparecerá un botón para desplegarse a otra ventana .

4.En esta ventana el usuario validará las últimas condiciones. Estas condiciones son: Modo de juego (si se hace atuomáticamente o el usuario decide que hacer), el tipo de ordenamietno (si el usuario decide llenarlo en modo pila o colo, esto si se hace automáticamente) y la dificultad (restringirá la contidad de movimientos que el usuario pueda hacer). Cuando los datos se ingrsen de manera correcta, podrá desplazarse a la pantalla del juego.

5.Una vez mostrada la pantalla principal se pueden dar dos casos, que el juego ya esté resuelto o que el usuario tenga que resolver, esto dependiendo de lo elegido anteriormente. Si el usuario decide que se ejecuta de manera automática se mostrará en pantalla el juego ya ordenado, mostrando en la pantalla las especificaciones que el usuario dió y permitiendo guardar el txt ya ordenado .

6. Por el otro lado, si el usuario decide realizar el juego de igual manera aparecerán las especificaciones que dió, así como una serie de opciones que le permitirán elegir el tipo de ordenamiento, y de que lista a que lista desea mover el color.

7. La pantalla principal cuenta con ciertas validadaciones, como por ejemplo que no se inserten más colores en una lista que la capacidad que tiene, o que no se extraiga cuando ya no hay elementos.

8.El usuario puede descargar la partida como la lleva actualmente en el momento que lo desee.

9.Así mismo, si el usuario gana le permitirá mostrar los pasos que realizó junto con el txt ordenado .
